What I have learnt during the process

  1. The thinking mode of MPLAB Harmony (separate everything )

  2. How to find "what I should do to learn something"

  3. MPLAB Harmony and MPLAB X IDE

  4. C style macro expansion and definitions

  5. General idea of transplanting library

    • Do not change the function implementation, but use #define lib_func(args) board_func(args) to change all the functions.

    • Write code for different MCUs

       
      x
      #ifdef __microcontrollerA__
      ...
      #endif

    • Change the dependencies destination

  6. The similarity of NFC chip, XBEE and web socket

    • Transparent mode and pass-through mode
    • Basic program can be as simple as a simple ISR of UART/I2C, while the difficulty lies in how to receive (length counting...)
    • Both have some configuration register, which is the reason why transplanting the library is a better choice than start from scratch.
